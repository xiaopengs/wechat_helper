# 小绿书创作 — 让信息在滑动中完成传递

## 目标

将结构化内容（教程、清单、对比、攻略、观点）转化为微信公众号「图片·文字」消息格式。每张图独立可传播，图文互补而非重复。

## 第一性原理

> **小绿书的本质不是「做漂亮的图」，而是「让信息在滑动中完成传递」。**
> 图是信息容器，文是注意力钩子。

核心公式：**70% 信息设计 + 20% 排版 + 10% 插画**（来源：AI 技术科普海报风格分析）

## 高风险操作黑名单

| 禁止操作 | 危害 | 替代方案 |
|---------|------|---------|
| 图片和文字内容重复表达 | 浪费读者注意力，图文1+1<1 | 图给数据/结构/对比，文给解读/钩子/引导 |
| 单页塞多个主题 | 信息过载，读者记不住任何一点 | 一页 = 一个信息点，宁可多一页 |
| 封面没有「点击理由」 | 0.1秒划走，前功尽弃 | 数字/悬念/利益点三者至少有其一 |
| 结尾没有行动引导 | 读者看完就关，无转化无传播 | 收藏/转发/评论，给读者一个下一步 |
| 用「在当今」「众所周知」「随着…的发展」开头 | AI腔暴露，信任瞬间归零 | 直接从具体数字、反常识、痛点场景切入 |
| 用非标准字体 | 微信不渲染自定义字体 → 排版崩坏 | 系统内置字体：HarmonyOS Sans SC / MiSans / OPPOSans → 文字转 PNG 锁定 |

## 编码规则

### 规则 1：每页一个信息原子，视觉占比 ≥30%

每张卡片必须有一个核心信息点（数字/结论/对比），在画面中占据至少 30% 面积。

- **字号标准**：主信息 48-72px（`HarmonyOS Sans SC Heavy`），正文 ≥ 16px
- **尺寸标准**：统一 1080×1440px（3:4），JPEG quality 85%，≤ 2MB
- **留白原则**：文字区域 ≤40% 画面，留白 ≥30%
- **生成工具**：`gpt-image-2`（封面/关键页，2K）→ `Pollinations.ai`（内页/批量，免费）→ `MiniMax image-01`（场景图）

> **失败机制**：字号 < 24px → 手机 375px 宽屏幕不可读 → 读者直接划走 → 完读率 < 15%。设计时必须用 375px 等效宽度模拟校验。

### 规则 2：钩子前置，前 3 句决定停留率

每页配文第一句必须是钩子。正文 50-200 字，每句 ≤ 25 字。

- **钩子类型**：具体数字（「73%」）、反常识（「你以为…其实…」）、痛点共鸣（「我也踩过这个坑」）
- **结构模板**：钩子句 → 展开句 → 结论/引导句
- **避免句式**：「在当今」「众所周知」「随着XX的发展」——这些是 AI 腔第一诊断信号

> **失败机制**：开头泛泛而谈 → 触发读者「这条没信息量」的潜意识判断 → 100ms 内划走 → 整篇白做。用「遮掉第一句」测试：如果从第二句开始读仍然成立，第一句就是废话。

### 规则 3：配色统一闭环，不许跳色

同一篇小绿书的所有页面必须同底色、同字体族、同排版架构。

- **配色上限**：主色系 1-2 个 + 强调色 1 个
- **预置方案**：6 套（暖橘/深海蓝/森林绿/薰衣草/极简黑白/AI科技蓝紫）→ `references/design-system.md`
- **科技类专用**：`#4F6BFF → #8B5CFF` 渐变 + 3D Isometric 插画 → `references/style-ai-tech.md`

> **失败机制**：不同页换配色 → 视觉碎片化 → 品牌感崩塌 → 读者看完记不住「这是谁发的」。统一不是审美偏好，是认知留存的手段。

## 失败机制速查

| 失败模式 | 触发条件 | 根因 | 症状 | 修复动作 |
|---------|---------|------|------|---------|
| 图片上传被拒 | 文件 > 2MB | 微信公众平台素材库上传限制 | 后台 toast：「文件大小不能超过 2M」 | `convert -quality 85%` JPEG 压缩，或 `pngquant` 减色 |
| 手机阅读不可用 | 字号 < 24px（生成时用 PC 参考） | 分辨率换算错误：PC 看起来大，手机屏物理像素少 | 文字模糊如蚂蚁，完读率暴跌 | 生成后缩小到 375px 宽目测，或 Prompt 加加「large bold text readable on mobile」 |
| 封面文字被裁切 | 图片比例 ≠ 3:4 | 微信图文消息封面自动居中裁剪为 1:1 或 3:4 | 标题上半段或下半段消失 | 封面必须 1080×1440，标题居中且占画面中上 30%-60% 区域 |
| AI 腔劝退读者 | 使用了论文/官文的模板句式 | 模型训练数据中安全表达占比高 | 完读率 < 20%，后台「分享」按钮点击为 0 | 检查每页首句 → 替换为数字/反问/痛点/具体例子 |
| 图文割裂 | 图说 A，文说 B | 图文分开构思，未联动 | 读者产生「这两张图有关系吗」的困惑 | 先写文案骨架，再根据文案定每页的图的内容；正文第 1 句必须与图中核心信息呼应 |
| 封面无法吸引点击 | 标题空泛无钩子 | 默认是「XX介绍」「XX指南」 | 公众号信息流中划过去无人停 | 标题加入：数字（「7个」）/ 反常识（「别再用XX」）/ 利益点（「省50小时」） |

## SOP

1. **选类型** — 从 5 种结构（清单/步骤/对比/反常识/地图）中确定一种 → 参考 `references/copy-templates.md`
2. **拆卡片** — 拆为 5-7 个信息原子，每卡含「核心视觉元素 + 50-200 字配文」
3. **选配色** — 从 6 套预置方案选一套 → `references/design-system.md`；科技类优先方案 6 → `references/style-ai-tech.md`
4. **生图** — `gpt-image-2` 生封面（重点），`Pollinations.ai` 生内页（批量），强制 1080×1440px
5. **验证发布** — 手机端预览 → 通过检查清单 → 公众号后台「图片/文字」→ 新建 → 发布

## 验证

- [ ] 所有图片 1080×1440px，≤ 2MB
- [ ] 封面标题占画面 30%-60%，375px 宽可读
- [ ] 每页只有 1 个核心信息点（遮掉其他文字仍成立）
- [ ] 每页第一句钩子测试通过（遮掉第一句后无影响 = 不合格）
- [ ] 配色全程统一（跨页对比：底色/字体/布局架构一致）
- [ ] 结尾有明确行动引导（收藏/转发/评论/点击）
- [ ] emoji ≤ 2 个/页，无 AI 腔模板句
- [ ] 已在手机端（375px 宽等效）预览
