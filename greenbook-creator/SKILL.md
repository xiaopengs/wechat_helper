# 小绿书创作指南 (Greenbook Creator)

## 概述

**小绿书** 是微信公众平台「图片·文字」消息格式的俗称，对标小红书笔记。核心理念：**图为主角，文为配角**——用高信息密度的图片承载核心内容，配合简洁有力的短文实现轻量化传播。

### 与公众号长文的关键区别

| 维度 | 公众号长文 | 小绿书 |
|------|-----------|--------|
| 内容载体 | 文字为主，图片为辅 | 图片为主，文字为辅 |
| 单篇字数 | 2000-5000字 | 500-1500字 |
| 阅读场景 | 深度阅读，PC/手机 | 碎片滑动，手机为主 |
| 信息密度 | 层层递进，逻辑链 | 单页闭环，每页独立 |
| 传播性 | 靠内容深度 | 靠视觉冲击+金句 |
| 制作成本 | 写作为主 | 设计+写作并重 |
| 适合内容 | 深度分析、教程、思考 | 攻略、清单、对比、展示 |

### 小绿书格式规格

- **封面图**: 3:4 或 1:1，推荐 1080×1440px
- **内页图**: 统一比例（推荐 3:4），推荐 1080×1440px
- **图片数量**: 2-9 张（含封面），最佳 5-7 张
- **每图配文**: 50-200 字，总正文 ≤ 1000 字
- **标题**: ≤ 64 字，封面醒目展示
- **发布渠道**: 微信公众平台 → 图片·文字消息

## 一、内容类型判断：什么适合做小绿书？

### ✅ 适合做小绿书的内容

- **步骤教程**: 每个步骤一张图，图文配讲解
- **清单盘点**: 书单/工具清单/资源合集
- **对比评测**: A vs B 逐维对比，每维一页
- **攻略指南**: 路线/景点/美食，每站一图
- **思维模型**: 一个模型一张图解
- **产品展示**: 场景图+核心卖点
- **穿搭/好物**: 单品展示+搭配说明
- **数据可视化**: 图表+关键解读
- **观点卡片**: 每页一个观点，适合碎片传播

### ❌ 不适合做小绿书的内容

- 需要完整逻辑论证的深度分析
- 大量代码/技术细节的教程
- 纯文字故事/叙事
- 需要引用大量外部链接的内容

## 二、创作流程

### 阶段 1: 选题策划

**1.1 确定内容类型**

从上面的「适合做小绿书」列表中找到对应类型。

**1.2 信息搜集**（新产品/概念必须执行）

- 搜索官方文档、产品介绍、用户评价
- 保存到 `_knowledge_base/[主题]-[日期].md`
- 确认理解后再进入下一步

**1.3 选题讨论**

提供 3-5 个选题方向，每个包含：
- 标题（吸引人、口语化）
- 核心角度（一句话切入姿势）
- 预估页数（2-7 页内页）
- 大纲（每页一个核心信息点）

### 阶段 2: 内容策划

**2.1 拆分信息卡片**

将内容拆分为独立的「卡片」：

```
卡片 1（封面）: 标题 + 核心悬念/价值承诺
卡片 2: 痛点/背景引入
卡片 3-N: 核心内容，每卡一个信息点
卡片 N+1（结尾）: 总结 + 行动引导/Call to Action
```

**2.2 每卡文案**

- 每张图配 50-200 字正文
- 正文要能独立阅读（考虑朋友圈转发场景）
- 关键信息用 emoji/符号标记
- 开头前 3 句决定停留率

### 阶段 3: 图片设计

**3.1 整体风格**

- 统一色调（建议 1-2 个主色 + 1 个强调色）
- 统一字体（标题 + 正文各一个字体族）
- 统一排版布局（左右/上下/居中）
- 保持品牌识别度（LOGO/水印/色系）

**3.2 封面设计原则**

- 标题占封面 30-40% 区域，字号要大
- 背景图不能喧宾夺主，留出文字空间
- 必须有「点击理由」——数字、悬念、利益点
- 推荐风格：马卡龙/渐变/毛玻璃/极简

**3.3 内页设计规范**

- 每页一个核心信息点
- 文字区域不超过图的 40%
- 关键数据放大、加粗、变色
- 留白充足，呼吸感
- 配色与封面保持统一

**3.4 AI 生图提示词模板**

```
风格：马卡龙美学，柔和渐变背景，圆角卡片，极简扁平设计
内容：[核心信息点和数据]
配色：[主色] + [辅色]，暖调/冷调
要求：留白充足，文字可读性高，信息层级清晰，适合手机竖屏阅读
```

### 阶段 4: 排版与发布

**4.1 图片顺序**

- 封面图（第 1 张）：标题醒目
- 正文图（第 2-N 张）：逻辑递进或并列
- 结尾图（最后 1 张）：总结 + 引导关注/转发

**4.2 正文排版规则**

- 每张图的配文用空行分隔
- 关键句加粗或用「」标注
- emoji 适度使用（每页 1-2 个）
- 避免大段文字，保持 3-5 句/段

**4.3 发布检查清单**

- [ ] 所有图片比例一致
- [ ] 封面标题清晰可读
- [ ] 每张图的信息层次分明
- [ ] 正文无错别字
- [ ] 图片顺序正确
- [ ] 封面预览效果 OK（手机端模拟）
- [ ] 已上传至公众号素材库
- [ ] 已在公众号后台预览确认

### 阶段 5: 发布后

- 记录发布数据（阅读量、分享量、完读率）
- 更新选题池状态
- 收集用户反馈用于迭代
- 在 `_drafts/` 归档终稿

## 四、文案风格规范

### 必须做到

1. 每页第一句就是钩子——让人想翻下一页
2. 用口语化短句，每句不超过 25 字
3. 数据要具体（"提升 37%" 而非 "大幅提升"）
4. 每页只讲一件事
5. 结尾必须有行动引导
6. 善用对比和反差制造冲击
7. 关键信息配 emoji 但不过量
8. 标题要有数字/悬念/利益点三者之一

### 必须避免

1. 大段文字堆砌（超过 5 行就拆）
2. 学术化/书面化表达
3. 空洞形容词（"非常好"、"特别棒"）
4. 每页讲多个主题
5. 结尾突然结束没有引导
6. 图片和文字内容重复（图文互补而非重复）
7. AI 味过重（"在当今…"、"众所周知…"）
8. emoji 密集轰炸

## 五、爆款公式

### 5 种小绿书爆款结构

**1. 清单型**（最高频）
```
封面: "X个必知的XXX"
P2: 为什么你需要这个清单
P3-P7: 每页一个条目（一句话+一图+一个理由）
P8: 总结 + 收藏引导
```

**2. 步骤型**
```
封面: "XXX的X步法"
P2: 我的失败经历 → 引出方法
P3-P7: Step 1-5，每步一页
P8: 效果展示 + 行动引导
```

**3. 对比型**
```
封面: "A vs B：到底选哪个？"
P2: 场景设定
P3-P7: 每维对比，左A右B
P8: 结论 + 你的选择是什么？（互动）
```

**4. 反常识型**
```
封面: "你以为XXX，其实XXX"
P2: 常见误解
P3-P6: 逐一打破，每页一个反常识
P7: 新认知总结 + 转发行
```

**5. 地图型**（攻略/资源）
```
封面: "XXX全景地图"
P2: 为什么要看全景
P3-P7: 每页一个区域/板块
P8: 下载/收藏引导
```

## 七、使用示例

### 示例：从一篇长文提炼小绿书

**功能说明**：将已有长文内容转化为小绿书格式

**执行流程**：
1. 读取长文，提取核心信息点（5-7 个）
2. 确定结构类型（清单/步骤/对比/反常识/地图）
3. 每个信息点 → 一页卡片（标题+配图描述+50-200字文案）
4. 设计封面（标题+悬念）
5. 生成图片 + 排版 + 发布

**关键要点**：
- 长文 5000 字 → 小绿书 7 页（含封面结尾）
- 删除论证过程，只保留结论
- 每页独立可传播
- 图片信息密度要大（一图胜千言）
