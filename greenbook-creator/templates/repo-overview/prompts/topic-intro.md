# 总导语模板（≤ 300 字，公众号「图片·文字」首图说明）

把以下变量代入即可。`{}` 是必填，`[]` 是可选。

```
你有没有发现，每次让 {职介动词} {情境}，{主体} 都得先花一半时间{重复动作}。
{痛点例证 1}，{痛点例证 2}，{痛点例证 3}——{两方} 都在重复同一个动作：{隐性代价}。

**{项目名}** {最近开源时间} 开源 —— 一个会替你{目标对象}{动作描述}的 CLI。
它{技术动作 1}、{技术动作 2}、{技术动作 3}，最后通过{交付方式}把更新交回来审。
这意味着：{一句收益 1}、{一句收益 2}、{一句收益 3}。

怎么开始？`{命令 1}`、`{命令 2}`、把官方 workflow 复制进 `.github/workflows/`，最快 N 分钟跑通。

下面 6 张图把它的痛点、相对竞品的差异、两种用法都拆给你看。⭐ {Stars}、{License} 开源、{Version}，建议先拿非核心仓库试点。
```

## 写作禁忌

- **不要**：在当今 / 众所周知 / 赋能 / 打造 / 引领
- **不要**：用"完美 / 极致 / 王炸"等过度营销词
- **要**：3-3-1 节奏（痛点 3 例 + 优势 3 句 + 行动 1 步）
- **要**：数字必有源（Stars / 时间 / 命令均标可验证）
- **要**：收尾给「低承诺免责」（"建议先非核心仓库试点"，避免被喷）
