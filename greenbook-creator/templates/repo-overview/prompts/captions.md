# 单图配文模板（P1–P6，每条 50–150 字）

每张图发出来时配的文字模板，按页码定位。

---

## P1 ｜封面
```
给 {Agent/角色} 助手一份会自动更新的项目百科。
封面一句话：**{项目名} = 持续 {动作} 的 {对象}**，不是又一个 {对立项}。
{License} 开源、{Version}、{Stars}。
```

---

## P2 ｜痛点
```
{Agent} 进陌生仓库的成本，不是调用 API，而是「重新认识这套代码」——
{痛点 A}、{痛点 B}、{痛点 C}。
{项目名} 把「重新读仓库」变成一条流水线：{动作 1} → {动作 2} → {动作 3}。
{一句话定性}。
```

---

## P3 ｜优势
```
真护城河不是「会{动词}文档」，而是「会跟着代码 {副动词} 文档」。
{优势 1}、{优势 2}、{优势 3}、{优势 4}。
{differentiator：换技术角度强调自己的不可替代点}。
```

---

## P4 ｜双模式
```
同一个 CLI，两条完全不同的知识生产线：
**{模式 A}** {写什么 A}，**{模式 B}** {写什么 B}。
共同底座是 {共用技术}，{凭证 / 数据来源说明}。
```

---

## P5 ｜竞品
```
别问谁最好，先看你要给谁写。
- **{项目名}** → {本项目一句话定位}
- **{竞品 1}** → {一句话定位}
- **{竞品 2}** → {一句话定位}
- **{竞品 3}** → {一句话定位}
{项目名} 的相对优势：{一句话数字 / 物理位置差}。
```

---

## P6 ｜上手
```
{数量} 条命令 N 分钟跑通：
1. `{命令 1}` — {一句话}
2. `{命令 2}` — {一句话}
3. `{命令 3}` — {一句话}
4. `{命令 4}` — {一句话}

{两个建议 + 一个免责}。

⚠️ {版本 / 试点提示}。
```

---

## 配套互动引导（结尾发）

```
你平时怎么{本题相关动作}？评论区告诉我，下一篇整理 N 种实战做法 📩
```
