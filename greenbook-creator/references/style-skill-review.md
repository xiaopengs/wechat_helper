# 工程师向「项目/技能速查」风格（Skill Review · 工程师盘点体）

> **核心公式**：「70% 信息密度 + 20% 实操步骤 + 10% 个人体感」
>
> 本质是 **「工程师视角的 README 翻译」**，不是营销盘点。读者带着"我要不要装 / 怎么选"的决策需求来，带着"装哪个 / 怎么装 / 踩什么坑"的可执行答案走。

**源头样本**：ConardLi《花园开源 4 个神级 Skills：做视频、写网页、生成图、查知识库，一篇讲完怎么用》（神器每日推送转载，2026-06）

---

## 一、风格定位

| 维度 | 取向 |
|---|---|
| **情绪基调** | 理性 · 克制冷感 · 第一人称亲测 · 不煽情 |
| **读者关系** | 同侪工程师视角（不是专家俯视，也不是博主种草） |
| **语言风格** | 短句为主，口语化点缀，结论先行 |
| **适合主题** | GitHub 项目盘点 / Skills / 框架 / 工具矩阵 / 多方案选型 / 兼容性测试 |
| **不适合主题** | 情绪话题、辟谣打脸、人物故事、政策解读、AI 腔科普 |
| **页数规划（短图文）** | **5–6 页**（封面 + 1 导语 + 4 个项目卡 / 兼容表 / 决策辅助） |
| **页数规划（长图文）** | **10–14 页**（cartographer 走向，每页一个 Skill + 1 页命令页 + 1 页决策辅助） |
| **出图引擎** | 短图文：gpt-image-2（tokenrouter） / 长图文：HTML + Playwright（cartographer 类） |

## 二、配色体系

### 主色（工程编辑感：浅色 + 黑文字 + 单点强调）
```
背景白:   #FAFAFA  （不是纯白 #FFFFFF）
纯黑文字: #1A1A1A
深灰副文字: #4A4A4A
分割线:  rgba(0,0,0,0.10)
```

### 强调色（单点高亮，克制使用）
```
强调蓝:    #0EA5A4   (项目编号 / 一行命令 / 关键数字)
警示橘:    #F59E0B   (踩坑/小毛病)
成功绿:    #10B981   (已验证标签)
红:        #EF4444   (极少用，仅限「不推荐」)
```

### 配色禁忌
- ❌ 不要渐变（除非是封面 hero 区 1 处）
- ❌ 不要饱和度 > 70% 的颜色（喧闹）
- ❌ 不要超过 2 个强调色（视觉分裂）
- ❌ 不要暖橘调（那是 orange-parenting 风格）

## 三、字体层级

```
章节序号:  18px monospace  (01 / 02 / 03,等宽,带半透明)
主标题:    56-72px HarmonyOS Sans SC Heavy
副标题:    28-36px Medium
正文:      18-22px Regular
命令块:    16-20px monospace (JetBrains Mono / Fira Code / SF Mono)
注释:      14px Regular
```

**核心规则**：命令路径 / 版本号 / 文件名一律用 monospace，正文用黑体。两套字体并用是工程师盘的视觉签名。

## 四、文体骨架（关键）

### 4.1 总章节结构（6 段固定骨架）

```
导语       (00 / 1 段精简, ≤120 字)
0X 速查    (01 / N 个对象, 每对象 6 件套)
0X 方式    (02 / N 种安装/使用方式, 表格)
0X 实操    (03 / 1 个端到端示例, 4-6 步)
0X 兼容性  (04 / 兼容矩阵, 表格)
0X 选哪个   (05 / 决策辅助, ≤6 行 Bullet)
```

**数字标记**：用 `01 / 02 / 03 / 04 / 05` 而非 `一、二、三、四、五` 或 `首先、其次`，避免 AI 味。

### 4.2 每个对象介绍的 **6 件套**（强制）

```
[对象名]：[一句话定位]
├── 做什么          （1-2 句定义，不超过 50 字）
├── 适用场景        （2-4 个角色/场景，不超过 30 字）
├── 硬卖点          （3-5 条 bullet，每条带 1-2 个具体数字）
├── 踩坑/小毛病      （1-3 条第一人体感点评，口语化）
├── 先装的人        （1 句目标人群，不超过 20 字）
└── 一行命令         （单行代码块，可复制）
```

**漏任何一件 = 不合格**，因为读者要的是「装不装/怎么装」全套决策信息。

### 4.3 数字密度铁律

- 每个卖点至少 1 个具体数字（"6 步工作流"、"25 个菜谱"、"1920×1080 固定舞台"）
- 不用"很多""若干""大量"，改"23 个""6 个""3-5 个"
- 不用"最新""几小时前"，改版本号 `v1.0.3` 或日期 `2026-06-18`
- 不用"差不多""大概""接近"，改绝对值

### 4.4 版本 / 路径精确铁律

```
# ❌ 错误
"安装到 Skills 目录"
"新版已经发布"
"在某个文件夹"

# ✅ 正确
"→ .claude/skills/web-design-engineer/"
"v1.0.3，GitHub Releases"
"~/.skills/<name>/"
```

路径必须有 `~/` 或 `./` 前缀，绝对路径；版本必须有 `vX.Y.Z` 格式。

### 4.5 口语点缀位置（克制使用）

每个对象最多 1 处第一人体感句，候选句式：

- "实话实说，这 Skill 我装的时候以为就是个 XX..."
- "用了一周发现我想错了"
- "三个小毛病也得说："
- "试下来最稳的是 X，Y 不太行"

不要全篇口语化（会变成亲测博主），工程师盘是「冷感 + 1-2 处亲测」交替。

### 4.6 决策辅助收尾（必备，替代金句）

```
# 不合格结尾
"以上就是 X 的介绍，希望对你有帮助"
"觉得有用记得点赞收藏"

# 合格结尾（决策表 + 信源）
按你手头的活儿对号入座：
  ▸ 做技术分享 → web-video-presentation
  ▸ 做产品页 → web-design-engineer
  ▸ 量产配图 → gpt-image-2
  ▸ 本地知识库 → kb-retriever

如果还拿不准，直接去仓库 README 确认。
```

**铁律**：不写"看完点个关注/转发给朋友"，信源导向而非社交索取。

## 五、出图规格（短图文 5-6 页）

| 页面 | 内容 | 视觉元素 |
|------|------|---------|
| **P1 封面** | 标题 + N 个对象名字 + 装饰 | 浅色底 + 大字号 + 序号横排 |
| **P2 导语** | 一句话定位 + 来源 | 浅色底 + 单段文字 + 来源小字 |
| **P3-P4 项目卡** | 每页 1 个对象，6 件套压成视觉块 | 浅色底 + 6 个色块或 6 行 bullet + 一行命令大字号 |
| **P5 安装/兼容** | 表格或决策矩阵 | 浅色底 + 等宽字表格 + 强调色高亮 1-2 行 |
| **P6 决策辅助** | 「按场景选哪个」 | 浅色底 + ▸ 箭头列表 + 末尾信源 |

**字体 vs 视觉占比**：文字 ≥ 60%（这风格是文字主导，不是图片主导），图片只是排版框架。

## 六、出图规格（长图文 10-14 页，走 cartographer 路线）

走 `style-cartographer.md` 的 12 页骨架，按以下重新映射：

| 标准页 | 工程师盘点版映射 |
|--------|----------------|
| P1 封面 | "N 个 X + 一篇讲完怎么用" |
| P2 痛点 | 「还在选 X？看完这 N 个就够」 |
| P3 定义 | 「工程师向 XX 是什么」 |
| P4 价值 | 「为什么值得盘点」（直入对比） |
| **P5-P8 速查** | **每个对象 1 页（硬卖点 + 命令）** |
| **P9 安装方式** | **「5 种安装方式」表格页** |
| **P10 实操** | **端到端 4 步配图** |
| **P11 兼容性** | **「6 种 Agent 兼容性」表格页** |
| P12 收束 | 「按场景选哪个 + 仓库地址」 |

如果对象超过 4 个，速查段每对象 2 页（卡 + 命令分页）；不要超过 14 页否则读者疲劳。

## 七、Prompt 体系（生图预设）

`scripts/styles/skill-review.txt` 的内核：

```
项目/技能速查风格，纯白底（#FAFAFA）+ 纯黑文字（#1A1A1A）+ 单点青色强调（#0EA5A4）。
工程编辑感，像技术博客的「项目卡片截图」或 README 卡片。
信息密度高：硬编码版本号（vX.Y.Z）、绝对路径（~/foo/bar）、精确数字。
大字号等宽代码展示一行命令（JetBrains Mono / Fira Code 风格）。
强调「工程师视角盘点」而非营销海报；不堆 emoji，不加营销文案。
1080×1440px，可视化边界为 ≤1px 分割线，无圆角，无投影，无 3D，无渐变。
```

**禁忌**：emoji 装饰（≤2 个/页）、暖色（橘红黄是 orange-parenting）、3D 等距插画（那是 style-ai-tech）、阴影、圆角 > 4px。

## 八、复制模板

### 模板 1：单个对象 6 件套

```
[项目名]：[一句话定位]

做什么：[定义，≤50 字，避开营销词]

适用场景：[角色 A] / [角色 B] / [场景 C]

硬卖点：
  · [N 个 X] — [具体能力]
  · [Y 类 Z] — [具体能力，含数字]
  · [W 项能力] — [具体能力]

踩坑/小毛病：
  · [问题 1，1 行]
  · [问题 2，1 行]

先装的人：[一句话目标人群]

一行命令：
  <[等宽字，单行]> npx skills add <repo> -s <skill-name>
```

### 模板 2：安装方式表格

```
[句号] / N 种安装方式，该选哪一种

| # | 方式 | 入口 | 适合谁 |
|---|------|------|--------|
| A | npx CLI | npx skills add <repo> | 默认推荐，自动识别 Agent |
| B | 插件市场 | /plugin marketplace add | 只用 Claude Code |
| C | Releases zip | GitHub Releases 下载 | CI、内网、版本锁 |
| D | git clone + cp | cp -r skills/<name> .claude/skills/ | 想 fork 自己改 |
| E | git submodule | git submodule add | 大型工程要 vendor |

[闪电 emoji，建议框]
拿不准的话先用 A。CI 和生产环境再换到 C 做版本固定。
```

### 模板 3：决策辅助

```
[场景 emoji] / 按场景先装哪一个

仓库地址：<github.com/xxx/yyy>，<LICENSE>，<可商用？>。
（注意：公众号会剥除外链，建议访问仓库查看。）

按你手头的活儿对号入座：
  ▸ [场景 1] → [对象 A]
  ▸ [场景 2] → [对象 B]
  ▸ [场景 3] → [对象 C]
  ▸ [场景 4] → [对象 D]

如果还拿不准，直接去仓库 README 确认：<地址>。
```

## 九、失败机制速查

| 失败模式 | 触发条件 | 根因 | 修复动作 |
|---------|---------|------|---------|
| **像营销盘点** | 通篇"重磅""神器""炸裂""绝绝子" | 缺版本/路径/数字硬度 | 替换为 v1.0.3 / ~/.skills/<name>/ / 23 个 |
| **缺决策辅助** | 结尾是金句/抒情/求点赞 | 缺少信源导向意识 | 改成"按场景选哪个 + 信源地址" |
| **6 件套缺 1** | 某个对象只写了"做什么"+"卖点" | 没意识到读者要做决策 | 严格 6 件套 + 自检清单跑一遍 |
| **段落均匀 AI 味** | 每段 3-4 行 + 章节编号一/二/三 | 模板复制过深 | 删章节编号改 01/02 + 加 1-2 个 1 行短段 |
| **数字模糊** | 出现"很多""最新""差不多" | 没强制数字密度 | 替换为绝对数字 / 版本号 / 日期 |
| **路径抽象** | 出现"安装目录""某个文件夹" | 没看过真仓库 | 改为绝对路径 `~/foo/bar` 或 `./.claude/skills/` |
| **强求互动** | 结尾"看完点个关注" | 跟抖音/小红书学多了 | 改为信源导向（仓库/文档/版本号） |
| **口语过多** | 每段都"我装了""我发现" | 误以为亲测 = 工程师视角 | 每对象最多 1 处口语点缀 |

## 十、自检清单（出稿前逐项跑）

- [ ] 总章节是否 6 段固定骨架（00 → 05）？
- [ ] 每个对象介绍是否完整覆盖 6 件套？
- [ ] 每个卖点是否带 1-2 个具体数字？
- [ ] 版本号是否为 `vX.Y.Z` 格式（不写"最新"）？
- [ ] 路径是否为绝对路径（`~/` 或 `./` 前缀）？
- [ ] 是否每对象最多 1 处第一人体感句？
- [ ] 章节序号是否用 `01/02/03` 而非 `一/二/三`？
- [ ] 结尾是否为决策辅助表 + 信源（非金句/非求互动）？
- [ ] 是否全文无"首先/其次/最后/在当今/众所周知"等 AI 腔？
- [ ] 出图：浅色底 + 黑文字 + 单点青色强调 + 等宽命令字？
- [ ] emoji 是否 ≤ 2 个/页（不是营销盘）？
