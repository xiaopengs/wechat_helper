# 小绿书文案模板

## 一、封面标题公式

### 公式 1：数字 + 名词 + 价值
- "7 个让你效率翻倍的 AI 工具"
- "2025 年必读的 5 本思维升级书"
- "3 步搞定朋友圈高赞排版"

### 公式 2：反常识 + 颠覆认知
- "你以为在省时间，其实在浪费生命"
- "90% 的人不知道的 XX 真相"
- "别再用 XX 了，试试这个"

### 公式 3：对比 + 选择
- "iPhone vs 小米：拍照到底谁强？"
- "大厂 vs 创业：3 年后我后悔了吗"
- "自学 vs 报班：我花了 2 万买来的教训"

### 公式 4：场景 + 痛点 + 方案
- "周末去哪玩？上海周边 5 个宝藏地"
- "写不出文案？试试这个万能框架"
- "换季不知道怎么穿？3 套公式直接抄"

### 公式 5：借势 + 热点
- "ChatGPT 更新后，这 3 个用法绝了"
- "看了《XX》才明白，原来…"
- "最近爆火的 XX，到底值不值得追？"

## 二、正文卡片文案模板

### 模板 A：清单条目式
```
[emoji] [条目名称]
[一行说明/理由]
[关键数据/金句]
```

示例：
```
🧠 费曼学习法
用"教给别人"来检验自己是否真懂
掌握率从 10% → 90%
```

### 模板 B：步骤引导式
```
Step [N]：[步骤名]

[做什么] → [为什么] → [效果]

💡 一个技巧：[具体可操作的建议]
```

示例：
```
Step 2：设定一个不可能失败的目标

每天只写 50 字 → 降低启动阻力 → 21 天后习惯自动运行

💡 一个技巧：把目标贴在刷牙的镜子上，每天刷牙时就能看到
```

### 模板 C：对比展示式
```
⚔️ 维度：[对比维度名]

🅰️ [选项A]：[一句话评价]
🅱️ [选项B]：[一句话评价]

🏆 这一轮：[结论]
```

示例：
```
⚔️ 上手难度

🅰️ Notion：功能多但学习曲线陡，新手容易懵
🅱️ 飞书文档：极简设计，打开就能写，零学习成本

🏆 这一轮：飞书胜，尤其适合团队协作新手
```

### 模板 D：观点卡片式
```
[核心金句/观点：14-20字]

[一行展开]

[一个具体例子/数据支撑]
```

示例：
```
AI 不会取代你，会用 AI 的人才会

工具从来都是放大器，不是替代品

就像 PS 没取代设计师，但会 PS 的设计师薪资高 2-3 倍
```

### 模板 E：数据冲击式
```
[大号数字]

[数字的含义]

[为什么这个数字重要]
```

示例：
```
73%

这是用 AI 后，我写一篇文章节省的时间比例

意味着以前一年写 50 篇，现在可以写 185 篇——深度和质量不变
```

## 三、开头与结尾

### 3 种开头方式

**悬念式**：
> "用了 3 年 Notion，我决定放弃了。不是因为它不好，而是我发现了一个更好的选择——而且它完全免费。"

**痛点共鸣式**：
> "做自媒体的第 100 天，我崩溃了。每天花 3 小时排版，阅读量还是两位数。直到我发现了这个…"

**价值承诺式**：
> "这篇文章能帮你省下 50 个小时的搜索时间。我整理了 2025 年最好用的 7 个 AI 工具，每一个都亲自测试过。"

### 3 种结尾方式

**行动引导式**：
> "你最想试哪一个？评论区告诉我，我私信你使用教程 📩"

**升华总结式**：
> "工具永远在变，但'先完成再完美'这个道理，大概永远不会过时。"

**互动转发式**：
> "如果觉得有用，转发给一个同样在焦虑的朋友。说不定你们能一起不那么焦虑一点 🤝"

## 四、常见内容类型的完整框架

### 类型 1：工具推荐
```
P1 封面: "N个[领域]神器，第X个绝了"
P2 痛点: "你是不是也遇到过[痛点场景]？"
P3-P7: 每页一个工具（名称+一句话亮点+适用场景）
P8 总结: "[场景]用[X]，[场景]用[Y]" + 收藏引导
```

### 类型 1B：工程师向工具/项目盘点（Skill Review · 适合 GitHub 项目转载 / Skills 速查 / 框架对比）

> **风格定位**：工程师视角的 README 翻译，不是营销盘点。情绪基调：理性 · 克制冷感 · 同侪亲测。适合转载 GitHub 项目 / 同主题多 Skill 对比 / 框架选型矩阵。详见 [`style-skill-review.md`](style-skill-review.md)。
>
> **完整章节骨架（6 段固定）**：
```
P1 封面    导语 + 来源标注（"ConardLi 在 GitHub 一次性放出 N 个 X，许可证、可商用"）
02 速查    N 个对象，每个对象必须含 6 件套：
           做什么 / 适用场景 / 硬卖点(带数字)/ 踩坑/小毛病 / 先装的人 / 一行命令
03 方式    N 种安装/使用方式对比表 + 默认推荐
04 实操    端到端 4-6 步示例（装命令→看落点→验证激活→锁版本）
05 兼容性  兼容矩阵表（Agent/工具/平台 × 功能是否可用）
06 选哪个  按场景决策辅助（▸ 4-6 个场景箭头列表）+ 信源地址
```

> **数字密度铁律**：每个卖点必带 1-2 个具体数字；版本号硬编码（v1.0.3 而非"最新"）；路径硬编码（~/.claude/skills/<name>/ 而非"安装目录"）。
>
> **口语点缀**：每对象最多 1 处第一人体感句（"实话实说，我装的时候以为...用了一周发现我想错了"），克制不滥用。
>
> **章节序号**：用 `01 / 02 / 03 / 04 / 05` 而非 `一、二、三、四、五`，避免 AI 味。
>
> **结尾禁忌**：不写"看完点个关注""觉得有用记得点赞收藏"，信源导向（仓库 README）替代社交索取。
>
> **出图风格**：浅色底（#FAFAFA）+ 黑文字（#1A1A1A）+ 青强调（#0EA5A4）+ 等宽命令字（JetBrains Mono），详见 [`style-skill-review.md` § 配色字体](../style-skill-review.md) 和 [`scripts/styles/skill-review.txt`](../scripts/styles/skill-review.txt)。

### 类型 2：读书笔记
```
P1 封面: "[书名]：一句话颠覆了我的认知"
P2 引子: "这本书回答了[什么问题]？"
P3-P7: 每页一个核心观点（金句+个人解读）
P8: 总结 + "你最近读了什么好书？"
```

### 类型 3：教程/攻略
```
P1 封面: "X步搞定[任务]，新手也能做"
P2 成果预览: "先给你看成品 → [效果图/数据]"
P3-P7: Step 1-5，每步配图
P8: 常见坑 + 避坑指南
```

### 类型 4：观点洞察
```
P1 封面: "关于[话题]，我想说点不一样的"
P2 共识/现状: "现在大家都在说[XXX]"
P3-P6: 每页一个反常识/新视角
P7: "所以我的结论是…"
P8: 开放式提问 + 引导讨论
```

## 五、文案自查清单

- [ ] 每页第一句能在 3 秒内抓住注意力？
- [ ] 读完每页只需要 15-30 秒？
- [ ] 有具体数字而不是模糊形容词？
- [ ] 每页只讲一件事？
- [ ] 有配图可以参考？
- [ ] 没有"在当今"、"众所周知"这类 AI 腔开头？
- [ ] emoji 不超过每页 2 个？
- [ ] 结尾有明确的行动引导？
