# 公众号账号诊断优化分析器（API版）

## 前置条件

**必须拥有**：认证服务号/企业订阅号的 `AppID` + `AppSecret`

⚠️ **个人订阅号无法使用数据API**，只能做基于单篇文章的手动分析

---

## 核心工作流（8步完整诊断）

### 第一步：凭证校验与Token获取

| 凭证获取方式 | 优先级 |
|------------|--------|
| 用户手动输入 --appid + --secret | 1 |
| 环境变量 `WECHAT_APPID` / `WECHAT_APPSECRET` | 2 |
| `~/.openclaw/wechat-secrets.json` 配置文件 | 3 |

**必须成功获取 `access_token` 才能继续**

---

### 第二步：自动拉取基础数据

执行 `scripts/account_analyzer.py fetch` 命令，自动拉取：

| 数据类型 | API接口 | 拉取范围 |
|---------|---------|---------|
| **历史文章列表** | `material/batchget_material` | 最近20篇图文素材 |
| **文章统计数据** | `datacube/getarticlesummary` | 最近30天（含阅读/在看/分享） |
| **用户增减数据** | `datacube/getusersummary` | 最近30天 |

**获取的字段**：
- 文章：标题、作者、摘要、发布日期、永久链接、封面图
- 统计：阅读人数/次数、分享人数/次数、收藏数
- 用户：新增关注、取消关注、净增关注

⚠️ **微信API限制说明**：
- `getarticlesummary` 只能获取**最近3个月**的数据
- 每次调用最多拉取**7天**数据，脚本会自动分批次拉取
- 不能获取单篇文章的**精确阅读数**，只能获取**按天汇总的图文数据**

---

### 第三步：基础数据指标计算

| 指标 | 计算公式 | 说明 |
|------|---------|------|
| 平均阅读量 | 总阅读 / 文章数 |  |
| 中位数阅读 | 排序后中间值 | 排除极端值，更真实 |
| 分享率 | 分享次数 / 总阅读次数 | 行业基准 2-3% |
| 发布频率 | 文章数 / 天数 | 建议周更≥2篇 |

---

### 第四步：内容风格深度分析

#### 4.1 标题风格分类

自动识别每篇文章标题类型：

| 类型 | 识别规则 | 点击率预期 |
|------|---------|-----------|
| 数字型 | 包含阿拉伯数字 | ⭐⭐⭐⭐⭐ 最高 |
| 悬念型 | 包含问号/感叹号/反问 | ⭐⭐⭐⭐ |
| 名人/产品型 | 包含知名公司/产品名 | ⭐⭐⭐⭐ |
| 观点型 | 包含"我""其实""不是"等 | ⭐⭐⭐ |
| 提问型 | 向读者提问 | ⭐⭐ |

#### 4.2 主题分布聚类

基于标题+摘要的关键词，自动识别账号核心主题：

| 主题分类（可扩展） | 关键词示例 |
|-------------------|-----------|
| 大模型 | GPT, Claude, LLM, 大模型 |
| Agent智能体 | Agent, 智能体, 工作流, 自动化 |
| 编程工具 | Cursor, Trae, Copilot, 代码 |
| 创业产品 | 创业, 产品, 商业化, 融资 |
| 行业观察 | 趋势, 观察, 行业, 未来, 分析 |
| 技术架构 | 架构, 设计, 系统, 技术, 工程 |

计算 **TOP3主题集中度**：
- > 90%：过度集中（有内容枯竭风险）
- 70-90%：健康聚焦（最佳区间）
- 50-70%：略有分散（建议收窄）
- < 50%：严重分散（必须做取舍）

#### 4.3 人味指数检测

| 信号类型 | 示例关键词 | 加分/减分 |
|---------|-----------|----------|
| AI信号（减分） | 首先/其次/再次/最后、总而言之、综上所述 | ❌ 减分 |
| 人味信号（加分） | 我/我的/我觉得、说实话、昨天/今天/刚才、测试了 | ✅ 加分 |

**人味指数** = 人味信号 / (人味信号 + AI信号) × 100
- > 70分：人味十足
- 50-70分：比较自然
- 30-50分：AI痕迹较重
- < 30分：AI感很强

---

### 第五步：账号画像评分（100分制）

| 维度 | 满分 | 评分规则 |
|------|------|---------|
| 更新规律性 | 25分 | 发布频率 × 20（每周2.5篇满分） |
| 标题风格统一性 | 25分 | 主导风格占比 × 25 |
| 主题集中度 | 25分 | TOP3主题集中度 / 4 |
| 人味指数 | 25分 | 人味指数 / 4 |

> **满分账号** = 规律更新 + 风格统一 + 主题聚焦 + 人味十足

---

### 第六步：热点关联策略分析（可选）

如果用户提供热点资讯路径（如 `../热点资讯追踪/`），自动加载：

1. 读取最近的 AI 哨兵晚报/晨报
2. 提取当期核心热点事件和趋势
3. 对比账号现有主题覆盖度
4. 输出热点跟进策略矩阵：

| 策略 | 条件 | 建议动作 |
|------|------|---------|
| 立即跟进 | 高热度 + 高契合 + 尚未覆盖 | 本周内产出1-2篇深度内容 |
| 深度挖掘 | 已覆盖但可以深挖 | 做系列专题或第二落点 |
| 长期布局 | 趋势性热点 | 做持续跟踪和深度研究 |
| 谨慎回避 | 与定位不符或高风险 | 不要盲目追热点 |

---

### 第七步：可执行优化建议输出

**禁止输出**："优化内容"、"提升质量"这类空话

**必须输出**：

| 维度 | 具体建议格式 |
|------|-------------|
| 内容策略 | "砍掉XX、YY主题，聚焦AAA、BBB" |
| 标题优化 | "增加数字型标题比例到50%以上，尝试'痛点+数字'组合" |
| 人味提升 | "增加第一人称视角，每篇至少有1个个人测试经历" |
| 发布节奏 | "固定在周二、周四晚8点发布，建立读者预期" |

---

### 第八步：生成完整诊断报告

报告输出路径：`诊断报告_{日期}.md`

报告结构：
1. 账号画像总评（100分制 + 四维度雷达）
2. 基础数据指标（阅读/分享/频率等）
3. 内容风格深度分析（标题/主题/人味）
4. 可执行优化建议（分点列出）
5. 热点关联策略（如有热点数据）
6. 后续行动清单（本周/本月/长期）

---

## 一键使用命令

```bash
# 方式1：完整诊断（推荐）
python scripts/account_analyzer.py full \
  --appid YOUR_APPID \
  --secret YOUR_SECRET \
  --output 诊断报告.md

# 方式2：先拉数据，再多次分析
python scripts/account_analyzer.py fetch --days 30 --output data.json
python scripts/account_analyzer.py analyze --data-file data.json --output 报告.md

# 方式3：结合热点分析
python scripts/account_analyzer.py full \
  --hotspot-path ../热点资讯追踪/ \
  --output 诊断报告_含热点策略.md
```

## 常见问题

**Q: 为什么拿不到单篇文章的精确阅读数？**
> 微信官方API的 `getarticlesummary` 返回的是**按天汇总的数据**，不是精确到单篇文章的。但可以通过发布日期做关联匹配，拿到近似值。

**Q: 个人订阅号没有API权限怎么办？**
> 可以手动录入数据：把每篇文章的标题、日期、阅读、在看录入JSON文件，然后用 `analyze` 命令分析。或者考虑升级为认证账号。

**Q: 数据安全吗？AppSecret会不会泄露？**
> 凭证只在本地使用，不会上传到任何第三方服务器。建议使用环境变量或本地配置文件，不要把明文写在命令行历史里。
