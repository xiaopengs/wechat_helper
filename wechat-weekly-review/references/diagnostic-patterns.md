# 诊断模式库（按 condition 索引）

**用法**：拿到 `data.json` 后，按以下 condition 列表**只加载命中的模式条目**，不要全读。每个模式独立成段，标题就是 condition。

---

## P1. 算法冷却 (Algorithm Cooling)

**Condition**: 连续 2+ 周 `source_breakdown.recommend.pct` 单调下降，且 `total_views` 同时下降。

**诊断**: 算法对新爆款作者有「一次性奖励窗口」（通常 1-2 周 1 万+ 推荐量）。窗口关闭后，单凭内容质量难以恢复到原水平。

**行动**:
- 立即**切换主题簇**（算法按主题簇冷却，不按账号）
- 砍产能到 1-2 篇/周（写越多算法越判定为内容农场）
- 寻找下一个「范式转移 + 强情绪」选题重新打开窗口

**判定阈值**:
- `recommend.pct` 周环比下降 ≥ 30 个百分点 = 强冷却信号
- `total_views` 下降 ≥ 70% = 算法已大幅降权

---

## P2. 主题簇疲劳 (Topic Cluster Fatigue)

**Condition**: 连续 2+ 周 `top_articles[0]` 标题同属一个主题（如都关于 AI Agent/AI 框架/某公司产品），且 `recommend.pct` 持续下降。

**诊断**: 算法对「同一作者同主题簇」分发意愿递减（避免用户信息流单一）。换主题是恢复推荐量的必要条件。

**行动**:
- 下一个爆款必须**主题簇外**（原 Agent 簇 → 切到 AI 商业化/AI 投资/AI 工作流）
- 频率上**不能「同一簇多篇连发」**（如 Agent Skill 周发 3 篇 → 算法降权）
- 测试新簇的「簇识别关键词」：换关键词可触发算法重新分类

**判定阈值**:
- 同一主题簇 ≥ 3 周 + recommend.pct < 30% = 簇已疲劳

---

## P3. 产能稀释 (Output Dilution)

**Condition**: 当周 `output.total_posts` ≥ 7，且 `traffic.total_views` / `output.total_posts` < 100。

**诊断**: 多产 ≠ 多流量。产能过高会稀释头部注意力、触发算法判定「内容农场」、降低每篇推荐量。

**行动**:
- 砍到 2-3 篇/周，重点保 1 篇质量
- 留 1 篇给「候选爆款选题」，其他用轻量内容补位
- 测算每篇边际收益，发现负收益就停

**判定阈值**:
- 平均 < 50 阅读/篇 + 总阅读 < 1000 = 强烈建议砍产能

---

## P4. 订阅低活跃 (Subscription Inactivity)

**Condition**: `subscription_msg.pct` < 15%（持续 2+ 周）。

**诊断**: 关注用户打开订阅栏的频率极低，账号对存量粉的「唤起能力」弱。问题通常在：推送时间不固定 / 开头钩子不够强 / 没有追更预期。

**行动**:
- 固定推送时间（如周二/周五 20:00）建立读者预期
- 测试开头 3 行的钩子（前 3 行决定打开率）
- 用「系列文」制造追更（如「Skills 系列」「Agent 系列」每周一篇）

**判定阈值**:
- 订阅占比 < 10% = 严重唤起问题
- 订阅占比 10-15% = 偏弱，需优化

---

## P5. 搜索依赖 (Search Dependency)

**Condition**: `search.pct` > 35%，且 `recommend.pct` < 30%。

**诊断**: 流量主要靠搜索发现，说明推荐算法没主动推。但搜索来源用户精准度高（主动搜的人意图明确）。

**行动**:
- 保搜索：标题押关键词（标题含「XX 封杀/发布/评测」类热词）
- **同时**赌推荐：选 1 篇/月做「范式转移」型选题（不押搜索，押情绪共鸣）
- 搜索流量高的内容如果爆，要立刻做「系列化」承接

**判定阈值**:
- search 占比 35-50% = 健康结构
- search 占比 > 50% = 严重依赖搜索流量，推荐能力弱

---

## P6. 单篇爆款孤峰 (Lone Peak)

**Condition**: `top_articles[0].shares` ≥ 5× 第二名，且 `top_articles[0].shares` ≥ 100。

**诊断**: 账号当前阶段靠 1 篇爆款撑场。其他文章没形成矩阵。爆款是一次性事件，不可复制。

**行动**:
- 爆款文末做「系列预告」承接流量
- 立刻分析爆款结构（用 wechat-explosive-analyzer）找可复用元素
- 但**不要追求复刻**—— 找下一个新簇选题重打

**判定阈值**:
- top1/total_shares > 50% = 单篇依赖严重

---

## P7. 粉丝增长停滞 (Follower Plateau)

**Condition**: `followers.net_change` ≤ 5，且持续 2+ 周；同时 `top_articles` 分享数较低（< 30）。

**诊断**: 内容没有「转发诱因」。读者愿意读但不愿转——通常因为：内容太专业（不易转）、观点太温和（无争议）、缺少「社交货币」属性。

**行动**:
- 标题加「社交货币」：争议性立场 / 圈内人才懂的梗 / 可截图的洞察
- 文中预留 5-8 条争议评论（让算法看到互动热度）
- 加「转粉钩子」：文末「关注看完整版」「下篇拆 X」

**判定阈值**:
- 连续 2 周 net_change ≤ 5 = 增长停滞
- 分享率 < 1.5% = 缺乏转发诱因

---

## P8. 头部空洞 (No Headliner)

**Condition**: 没有一篇 `total_views` > 500，且 `top_articles[0].shares` < 20。

**诊断**: 整周没有破圈内容。所有文章都「自给自足」但不爆。说明选题没有抓到情绪共鸣点或没押中热点。

**行动**:
- 选题必须挂「情绪钩子」：恐惧/愤怒/惊喜/优越感之一
- 标题必须含「反共识 / 强对比 / 数字 / 时间紧迫感」之一
- 找 3 个「上周全网讨论但你还没写」的热点，48 小时内出文

## 模式组合判定

实际诊断往往是多个模式叠加。常见组合：

| 组合 | 真实问题 | 优先动作 |
|---|---|---|
| P1 + P2 + P3 | 算法冷却 + 簇疲劳 + 产能稀释 | 砍产能 + 换簇 + 等新爆款 |
| P4 + P7 | 订阅弱 + 增长停滞 | 改造推送节奏 + 加社交货币 |
| P5 + P8 | 搜索依赖 + 头部空洞 | 押 1 篇范式转移型 + 其余搜索友好型 |
| P6 + P7 | 单篇爆 + 涨粉弱 | 爆款承接 + 文末钩子 |
