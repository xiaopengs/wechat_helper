# 跨周趋势方法论

**触发条件**：用户一次提供 2+ 周周报，或要求"对比上周 / 最近几周 / 月度趋势"。

## 第一步：建立趋势表

把多周 `data.json` 的核心指标抽成一张对照表。**必含字段**：

| 字段 | 用途 |
|---|---|
| `week_range` | 周次标识 |
| `total_views` | 看绝对增长/衰退 |
| `recommend.pct` | **算法推荐占比**，最关键的领先指标 |
| `search.pct` | 搜索依赖度 |
| `subscription_msg.pct` | 存量粉活跃度 |
| `top1_views` / `top1_shares` | 头部文章表现 |
| `net_change` | 净增粉 |
| `output.total_posts` | 产能（对比流量看效率） |

**计算衍生指标**：
- `avg_views_per_post` = total_views / total_posts
- `share_rate` = total_shares / total_views × 100%
- `wow_views_pct` = (本周 - 上周) / 上周 × 100%

## 第二步：识别 4 种趋势形态

### T1. 爆发-衰退型（典型新账号成长曲线）
- 形态：第 1 周巨量 → 2-4 周内断崖
- 识别：`recommend.pct` 从 ≥ 80% → 30% 以下；`total_views` 下降 ≥ 70%
- 含义：算法「一次性奖励窗口」已关闭
- 行动：见 P1（算法冷却）+ P2（主题簇疲劳）

### T2. 稳定低位型
- 形态：连续多周 `total_views` < 500，无显著波动
- 识别：周环比变化 < ±20%
- 含义：账号进入「稳定但小」阶段，无爆款、无衰退
- 行动：找新切口打破天花板（换主题簇 or 升级内容深度）

### T3. 复苏型
- 形态：连续低位 → 单周爆发（`total_views` 增长 ≥ 5×）
- 识别：前 3 周平均 < 500，本周 ≥ 2000
- 含义：新簇/新选题命中算法
- 行动：立刻**承接**—— 做系列、做簇内深挖、保产能稳给

### T4. 持续增长型（最难达成）
- 形态：连续 4+ 周周环比 ≥ +20%，且无大幅回撤
- 识别：`wow_views_pct` ≥ +20% 持续
- 含义：账号找到稳定的内容方法论 + 算法进入常态推荐
- 行动：保节奏 + 测试扩展（横展选题矩阵）

## 第三步：交叉信号分析

除了主形态，找**跨周信号异常**：

| 信号 | 含义 |
|---|---|
| `recommend.pct` 降 + `search.pct` 升 | 算法主动降权，但搜索 SEO 仍生效 |
| `subscription_msg.pct` 单周翻倍 | 出现一篇「追更型」内容，存量粉被唤醒 |
| `top1_shares` 暴涨但 `total_views` 平 | 爆款有传播但没破圈——可能进了窄圈 |
| `net_change` 持续为负 | 取关 > 关注，内容触怒或无人新进 |
| `output` 增 + `avg_views_per_post` 减 | 产能稀释（见 P3） |

## 第四步：输出报告结构

跨周报告 = 「趋势表 + 主形态判定 + 异常信号 + 行动清单」

```
# 趋势报告 YYYY-MM~YYYY-MM

## 一、核心指标对照
[趋势表]

## 二、主形态判定
[4 种形态之一 + 理由]

## 三、异常信号
[交叉信号列表]

## 四、行动清单
- 本周必做：[1-3 条最紧急]
- 本月目标：[2-3 条中期]
- 持续观察：[信号列表]
```

## 加载策略

- 跨周时**只加载本文件** + `weekly-data-schema.md` + 相关 pattern 条目
- **不要**把所有历史的 `diagnosis.md` 加载进 context
- 需要历史细节时用 grep 从 `review/weekly/*/data.json` 检索
