# 外部平台手动出图指南

当未配置 API Key 或选择手动出图时，请按以下步骤操作。

---

## 通用操作步骤

1. **打开 prompt.txt**：找到 Agent 生成的 prompt 文件路径（如 `$DIR_PATH/anim_01/prompt.txt`）
2. **复制完整内容**：将 prompt 全文复制
3. **前往选择的平台**：千问 / 豆包 / Gemini
4. **粘贴并生成**：在输入框粘贴 prompt，等待生成
5. **下载图片**：选择最满意的宫格图，下载保存
6. **返回给 Agent**：在对话中发送/粘贴图片

---

## 平台操作指南

### 阿里云千问

**地址**：https://tongyi.aliyun.com/

**操作步骤**：

1. 登录阿里云账号
2. 进入千问对话页面
3. 粘贴 prompt 到输入框
4. 等待图片生成
5. 点击下载保存

**特点**：国内访问稳定，响应速度快

---

### 字节豆包

**地址**：https://www.doubao.com/

**操作步骤**：

1. 登录/注册账号
2. 选择图像生成功能
3. 粘贴 prompt
4. 等待生成并下载

**特点**：中文场景效果好

---

### Google Gemini

**地址**：https://gemini.google.com/

**操作步骤**：

1. 登录 Google 账号
2. 进入 Gemini 对话
3. 粘贴 prompt
4. 等待生成并下载

**特点**：效果最佳，但可能需要国外网络

## 有定妆图时的操作 ⭐

**定妆图概念**：

- 白底、正面、无动作的角色基准图
- 用于锁定角色外观一致性，让每次生成的角色长相一样
- Agent 会在 `$DIR_PATH/base_reference.png` 保存定妆图

---

### 如果 Agent 提供了定妆图

**通用流程**：先上传定妆图 → 再粘贴 prompt → 确保平台识别到"图片 + prompt"的组合

---

### 千问 + 定妆图

1. 在输入框先上传 `$DIR_PATH/base_reference.png`
2. 再粘贴 prompt 文本
3. 确保平台识别到"图片 + prompt"的组合输入
4. 生成的图片会保持角色一致性

---

### 豆包 + 定妆图

1. 选择"图片参考"功能
2. 上传 `$DIR_PATH/base_reference.png`
3. 粘贴 prompt
4. 等待生成

---

### Gemini + 定妆图

1. 在对话框点击上传图片按钮
2. 选择 `$DIR_PATH/base_reference.png`
3. 粘贴 prompt
4. Gemini 会"看着"定妆图来生成，角色外观保持一致

---

**⚠️ 重要提示**：

| 操作                       | 结果                    |
| -------------------------- | ----------------------- |
| **每次都上传定妆图**       | 角色外观一致 ✅         |
| **忘记上传定妆图**         | 角色外观每次都不一样 ❌ |
| **只粘贴 prompt 不上传图** | 角色外观随机 ❌         |

- 定妆图的作用是让 AI "看着"图片画，保证角色长相一致
- 每次生图都必须带上**同一张**定妆图
- 定妆图路径：`$DIR_PATH/base_reference.png`（Agent 会告知你）

---

## 批量生成场景

如果需要生成多个表情（如 24 个 GIF）：

1. Agent 会生成多个 prompt 文件（anim_01/prompt.txt, anim_02/prompt.txt, ...）
2. 按顺序逐个操作：
   - 打开 anim_01/prompt.txt → 去平台生成 → 返回图片给 Agent
   - Agent 处理完成后 → 打开 anim_02/prompt.txt → 继续
3. Agent 会实时反馈进度："第 1 张完成"、"第 2 张完成"...

---

## 常见问题

**Q：prompt 太长，平台不支持？**
A：部分平台有长度限制，可尝试分段或简化描述

**Q：生成的图片不满意？**
A：可以多次生成选择最满意的，或在对话中告诉 Agent 调整 prompt

**Q：忘记上传定妆图？**
A：角色外观会不一致，建议重新生成并带上 `$DIR_PATH/base_reference.png`
