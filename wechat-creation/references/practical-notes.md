## 创作执行规范

### 结构公式
```
「故事/引入」→「深挖/对比」→「洞察/升华」
三幕式，阅读时长 6-8 分钟

开头：
- 具体时间/地点/事件/人物 → "上周..." "昨晚..."
- 自问自答驱动
- 口语化 → "对..."、"说起来..."

中段：
- 自问自答继续驱动节奏
- Cherry Studio结构 → 故事→洞察→启示（每个小节）
- DeepSeek文章模式 → 引入问题→故事一→故事二→回归问题→深层解析+理性表达

结尾：
- 回归开头的问题或主题
- 简洁有力，1-2句即可

金句公式：
- 「抽象概念 对 抽象概念」→ "便利的代价是想象力的丧失"
- 「数字 对于 人来说...」→ "在一个人人都能编程的世界里，程序员的角色正在被重新定义"
- 「AI 帮你...但不会替你...」→ "AI可以替我写，但不能替我想"
```

### 🟢 做得好的

| 手法 | 文中位置 | 效果 |
|---|---|---|
| 数字冲击 | 开头「78,943颗星，单日涨3,392」 | 3秒抓人，不可绕过 |
| 自问自答 | 「为什么是现在？」「谁会赢？」 | 驱动阅读节奏 |
| 类比增厚 | npm 2015 → Agent Skills 2026 | 提供认知坐标系 |
| 四层格局 | 框架→库→基础设施→市场 | 展示"想清楚了" |
| 连续引用 | 引用自己前文「AI可以替我写…」 | 建立内容人格 |
| 金句收尾 | 「标准化Skill可以替你积累，但不能替你判断」 | 留记忆锚点 |
| 评论区提问 | 四个具体问题 | 引导互动 |

### 🟡 可改进的

| 问题 | 位置 | 改进方向 |
|---|---|---|
| 中段项目列举密度偏高 | 「不只pocock」段 | 可减少并列项目数，选最有代表性的2-3个 |

## 二、jerry风格落地检查清单

### 开头检查

- [x] 不要「今天我们来介绍…」→ 用「前几天在GitHub Trending上刷到一个数字」
- [x] 第一段 < 100 字
- [x] 有具体数字或具体场景
- [x] 展示情绪反应（「把我惊了一下」）

### 叙事节奏检查

- [x] 每 2-3 段有一个自问自答（「为什么是现在？」）
- [x] 展示思考过程而非直接给结论
- [x] 技术深挖不超过 2 段就拉回主线
- [x] 段长 2-4 句，不超过 5 句

- [x] 具体数字：78,943 / 3,392 / 50K / 300K

- [x] 具体时间：2015 / 2014 / 2016

### 结尾检查

- [x] 有个人判断（「我的判断：最先成为默认选择的那个人赢」）

## 三、微信 HTML 排版要点速查

### 必须遵守


```
✅ 必须做：
- 所有样式 inline（style="..."）
- 只用 section / span / strong / em / br / img / a
- 正文 #333, 主色 #1a73e8, 强调 #ff6b35
- 背景 #ffffff, 字号 15px, 行高 1.8
- 分割线高度 1px, 颜色 #eeeeee
- 小标题：font-size: 18px, font-weight: bold, 左侧3px蓝色竖线, no text-align:center
- 金句：左侧3px蓝色竖线, #f7f8fa 灰色背景, 圆角 0 6px 6px 0

❌ 禁止：
- ❌ class 选择器（微信会吞掉）
- ❌ flex / grid / shadow / border-radius:(除金句)
- ❌ font-size 超过 18px（微信不生效）
- ❌ 斜体 em（微信不显示）
- ❌ 深色背景 (background-color: dark)
- ❌ Markdown 格式链接 `[...](...)`
- ❌ table 元素
- ❌ img 标签内的 width/height/html 属性
```

## 语言风格

### 通用口头禅
- "对..." "说起来..." "说实话..." "我想说..." "可能..."
- 避免 "综上所述" "因此" 等书面语

### 情感表达
- 通过口语化表达展现情感
- 用句号分隔短句，增强节奏感
- "多快？" "多厉害？" 自问自答

## 引用规范
- 用 Apple 风格引用格式：右上角数字角标（如 "¹"、"²"）
- 文末注明来源 + 链接，方便读者在一个地方找到所有引用

## 配图规范

### 配图原则
- ✅ 只用：架构图、技术方案对比图、能力对比图、知识概念图
- ❌ 不用：标题党封面、噱头插画、与内容无关的配图
- 每篇文章配图不超过 3 张（不含封面）
- 封面使用文章核心概念的视觉化表达

### AI图片生成技能迭代
- 代码抽离到 wechat-illustration 脚本
- 主力：Pollinations.ai (free curl)
- 备选：Minimax image-01
- 所有图片本地存到 `images/` 目录
