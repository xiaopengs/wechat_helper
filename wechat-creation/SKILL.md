# 公众号内容创作指南

## 任务目标

本 Skill 用于：为jerry的公众号内容提供全流程创作辅助

能力包含：
- 风格规范指导（10条必须做到 + 10条必须避免）
- 选题讨论（提供3-4个选题方向及大纲）
- 内容创作（初稿撰写 + 三遍审校）
- 爆款结构优化（5种爆文公式 + 完读率关键）
- 文件组织管理（brief、知识库、协作文档）

触发条件：
- 需要创作公众号内容
- 需要讨论选题方向
- 需要审校优化已有内容

## 前置准备

- 风格规范：阅读 [references/style-guide.md](references/style-guide.md) 了解jerry公众号风格要求
- 模板参考：阅读 [references/templates.md](references/templates.md) 获取各类模板（选题讨论、协作文档、长文转X）
- 文件结构：按需创建目录（`_briefs/`、`_knowledge_base/`、`_协作文档/`、`草稿/`、`images/`）

## 操作步骤

### Step 1: 保存需求Brief

- 保存创作需求到 `_briefs/[项目名]-需求brief.md`
- 明确以下信息：
  - 目标受众
  - 核心信息
  - 字数要求
  - 特殊要求（如有）

### Step 2: 信息搜索（⭐必须优先执行）

**强制规则**：当用户提出新的产品/技术/概念时，必须先完成信息搜索，然后才能进行选题讨论。跳过此步骤直接选题是错误的流程。

- 对于新产品/新技术，必须进行信息搜索
- 搜索内容包括：官方文档、产品介绍、技术特性、用户评价、竞品对比
- 保存搜索结果到 `_knowledge_base/[主题]-[时间].md`
- 分类存储：技术工具/行业人物/产品发布/报告资料/方法论
- 搜索完成确认：向用户确认搜索结果后再进入选题讨论

### Step 3: 选题讨论（必做）

**前置条件**：必须已完成Step 2信息搜索，对新概念/产品有充分了解后才能开始选题讨论。

提供3-5个选题方向，每个包含：
- 标题（吸引人、口语化，符合jerry风格）
- 核心角度（一句话说清切入姿势)
- 工作量评估（⭐评级）
- 大纲（3-7个大标题）
- 优劣势分析

### Step 4: 创建协作文档（如需测试）

- 保存到 `_协作文档/[项目名]-协作清单.md`
- 包含：
  - 测试任务清单

  - 时间成本预估

### Step 5: 学习风格

- 阅读 `../写作参考/` 文件夹中的参考文档
- **从 [assets/](assets/) 目录中随机选择2-3篇jerry的历史文章进行阅读**
- 理解jerry风格的关键特征

### Step 6: 使用个人素材库

- 加入真实经历降低AI味
- 引用具体的时间、地点、人物、事件
- 保持内容的真实感和可信度

### Step 7: 创作初稿

- 基于真实数据和搜索信息
- 保持「实践+落地」调性
- 初稿可不完美，后续审校环节优化

### Step 8: 三遍审校

- 第一遍：检查基本错误（错别字、标点、格式）
- 第二遍：优化表达和逻辑
- 第三遍：风格检查和内容升华

### Step 9: 爆款结构优化

**5种爆文公式**：
1. 问题-原因-方案-升华（教程类）
2. 故事-冲突-转折-洞察（经验类）
3. 颠覆认知-新视角-重建（观点类）
4. 对比-反差-原因-启示（对比类）
5. 数据冲击-深挖-案例-升华（分析类）

**完读率关键检查**：
- 开头（前100字）：3秒抓住、悬念设置、价值承诺
- 中段：每300字有新洞察、2-3个转折
- 结尾：升华主题、行动引导

### Step 10: 更新选题池

**文章发布后必须执行此步骤。**

选题池位于 `projects/wechat_helper/选题池/pool.json`，记录所有选题的生命周期状态。

**操作流程：**

1. **勾掉已发布选题**：将对应 topic 的 `status` 从 `"draft"` 或 `"selected"` 改为 `"published"`，补充 `selectedAt`、`publishedAt`、`wechatDraftMediaId` 字段
2. **新选题入库**：信息搜索完成后，将新发现的选题方向追加到 `pool.json` 的 `topics` 数组
3. **选题筛选**：选题讨论时，从池中挑选 `status: "draft"` 的条目录入 brief
4. **状态流转**：draft → selected（选题讨论选中）→ published（文章发布）；draft 且未选中保持原状态
5. **更新 updatedAt**：每次修改 `pool.json` 后更新顶层 `updatedAt` 时间戳
6. **提交推送**：所有变更提交到 `projects/wechat_helper` 仓库，推送到 GitHub

**选题池条目格式：**
```json
{
  "id": "topic-YYYYMMDD-NNN",
  "title": "文章标题",
  "angle": "一句话核心切入角度",
  "outline": ["大纲条目1", "大纲条目2", ...],
  "source": "信息来源",
  "difficulty": "★",
  "status": "draft|selected|published",
  "createdAt": "ISO时间戳",
  "selectedAt": "ISO时间戳或null",
  "publishedAt": "ISO时间戳或null",
  "wechatDraftMediaId": "微信草稿media_id或null"
}
```

### Step 11: 长文转X（可选）

**标准结构**（200-500字）：
1. 开头（30-50字）：金句/数据/价值主张
2. 展开（100-300字）：具体细节、个人经历
3. 结尾（20-30字）：互动引导

**X长文版**（1000字+）：
- 用"——"分隔章节
- 重要人名第一次@账号
- 结尾开放式问题

## 注意事项

- 仅在需要时读取参考，保持上下文简洁
- **流程顺序强制**：遇到新概念/产品，必须先执行Step 2信息搜索，再执行Step 3选题讨论，严禁跳过搜索直接选题
- 不要直接开始写文章，必须先进行选题讨论
- 严格遵守风格规范，避免AI腔
- 每篇文章都要进行三遍审校
- 充分利用智能体的语言理解和创作能力，所有环节由智能体指导完成

## 使用示例

### 示例1：个人文章创作

**功能说明**：创作个人经验分享类文章

**执行方式**：
- 智能体指导选题 → 结合个人经历和行业观察
- 智能体创作 → 使用心路历程式叙事、自问自答式叙事
- 智能体优化 → 应用爆款结构公式

**关键要点**：
- 加入具体的时间、地点、人物
- 展示决策过程（犹豫→思考→决定）
- 数据后加个人判断（"这个数据还挺猛"）

### 示例2：审校优化

**功能说明**：对已有内容进行审校和优化

**执行方式**：
- 智能体三遍审校 → 基本错误、表达优化、风格检查
- 智能体应用爆款公式 → 选择合适的结构框架
- 智能体检查完读率 → 开头抓人、中段转折、结尾升华

**关键要点**：
- 开头前100字要有悬念或价值承诺
- 中段每300字要有新洞察
- 所有案例都要有链接，重要内容标注[图片]
