# 公众号选题集成 Follow-Builders 信息源

## 集成架构

### 信息搜索增强 (Step 2)

原有流程中，信息搜索步骤聚焦于「用户给出的具体产品/技术/概念」。
集成后，**额外增加** Follow-Builders 的信息源拉取，作为行业趋势的同步信号。

### 搜索源

| 源 | 说明 | 用途 |
|---|---|---|
| `follow-builders/feed-x.json` | 25位 AI Builder 的 X 动态 | 发现AI行业热点、新产品、技术趋势 |
| `follow-builders/feed-podcasts.json` | 5个顶级AI播客 | 深度话题、行业洞察 |
| `web_search` | 传统网络搜索 | 具体概念/产品的深度调研 |
| `china-hot-ranks` | 国内热榜 | 国内用户关注的热门话题 |

### 执行流程

```
每日 8:00 AM (cron)
  │
  ├─ 1. 拉取 follow-builders 最新 feed
  │     node prepare-digest.js
  │     获取 AI Builders 最新动态 + 播客内容
  │
  ├─ 2. 识别热门趋势
  │     从 Builder 动态中提取：
  │     - 新产品/新功能发布
  │     - 技术争议/讨论热点
  │     - 行业趋势信号
  │     - 开源项目动态
  │
  ├─ 3. 生成选题建议 (3-5个)
  │     每个包含：标题、角度、大纲、工作量评估
  │     标注信息源（follow-builders / web / hot-ranks）
  │
  ├─ 4. 存入选题池 pool.json
  │     新选题状态 = "draft"
  │
  ├─ 5. 从选题池中筛选（排除已发布）
  │     优先选：信息时效性高 + 角度独特 + 工作量适中
  │
  ├─ 6. 生成当日选题记录
  │     保存到 选题池/YYYY-MM-DD.md
  │     更新 pool.json 中被选题状态 → "selected"
  │
  ├─ 7. 推送选题池全貌报告（无偏见）
  │     包含：池子总览 + 今日入选详情 + 完整待选池评估
  │     每个选题给出优点和风险，不美化，不偏袒
  │
  └─ 8. 等待用户确认 → 进入创作流程
       初稿 → 审校 → 爆款优化 → 发布到草稿箱
```

### 关键规则

1. **从不重复选取已选题**：pool.json 中 status="published" 的永不再次选择
2. **每日重新筛选**：不从上次的选择延续，每次都从整个未发布池中重新评估
3. **时效性优先**：follow-builders 拉取的最新动态优先于老旧选题
4. **选题存活期**：draft 状态的选题超过 14 天自动标记为 "expired"
