## 动态页数控制规则(SKILL.md 第 1 步详解)

### 公式

```
核心字数 CW = 知识背景中文字数(不含标点/空格/代码块)
必选页数 = 4 (CW < 80) 或 5 (CW ≥ 80,加 M2 痛点)
扩展页数 = clamp(round(CW / 150), 0, 8)
总页数 = min(必选 + 扩展, 12)
```

| 知识背景字数 | 必选 | 扩展 | 总页数 |
|----|----|----|----|
| < 80 字 | 4 | 0 | **4 页** |
| 80-150 字 | 5 | 0 | **5 页** |
| 150-300 字 | 5 | 1 | **6 页** |
| 300-450 字 | 5 | 2 | **7 页** |
| 450-600 字 | 5 | 3 | **8 页** |
| 600-900 字 | 5 | 4 | **9 页** |
| 900-1200 字 | 5 | 5 | **10 页** |
| 1200-1500 字 | 5 | 6 | **11 页** |
| 1500-2000 字 | 5 | 7 | **12 页** |
| ≥ 2000 字 | 5 | 8(钳) | **12 页**(提示拆套) |

### 扩展模块与字数的对应关系

| CW 范围 | 命中的扩展模块 |
|----|----|
| < 150 | 无(纯骨架) |
| 150-400 | M8 设计原则(三条) / M6 结构拆解(一句话级) |
| 400-800 | M6 + M8,若含代码命中 M7,若含案例命中 M9 |
| 800-1500 | M6 + M7 + M8 + M9 |
| 1500-2000 | + M10 概念图谱 |
| ≥ 2000 | + M11 Q&A,**提示拆套** |

## 字数计算的实现提示

agent 收到用户输入时,先做以下预处理:

```python
import re

def count_chinese(text):
    """只统计中文字数,排除标点/空格/英数/代码"""
    chinese_chars = re.findall(r'[\u4e00-\u9fff]', text)
    return len(chinese_chars)

# 示例
background = "OpenRouter 把 300+ 模型的 API 统一成一把 Key..."  # 用户输入
CW = count_chinese(background)
print(f"CW = {CW}")  # 输出 CW = 47
```

**注意**:
- **不算标点**(逗号 / 句号 / 引号)
- **不算空格 / 换行**
- **不算英文字母 / 数字**(代码块里的英文不算)
- **代码块本身不计入 CW**(代码是结构展示,不消耗"内容密度"预算)

## 页数上限处理

| 总页数 | 处理 |
|----|----|
| < 4 | 强制凑到 4(M1 + M3 + M4 + M5) |
| 4-12 | 正常输出 |
| > 12 | **拆套**:在输出合同里提示"建议拆为 [A] [B] 两套,每套不超过 12 页" |

## 模块顺序建议(12 页标准)

```
1.  M1 封面
2.  M2 痛点(CW ≥ 80)
3.  M3 定义
4.  M6 结构拆解 / M7 代码示例(按特征)
5.  M8 设计原则
6.  M9 落地样包
7.  M10 概念图谱(若宏大)
8.  M11 Q&A(若有)
9.  M4 价值
10. M5 收束
```

不必严格按此顺序,但 M1 一定在首,M5 一定在末。
